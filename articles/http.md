## 一言で言うと
クライアントとサーバーが Web 上でデータをやり取りするための通信規約（プロトコル）。

## 問い（復習時にここだけ見て答える）
- HTTP とは何か？
- HTTP のステートレスとはどういう意味か？
- ステートレスなのにログイン状態が保てるのはなぜか？

## ノート
- HTTP = Hyper Text **Transfer** Protocol（Transform ではない）
- クライアント（ブラウザなど）とサーバーが「共通のルール」で話すための仕組み
- 1回のやり取りは **リクエスト（要求）→ レスポンス（返却）** の一往復
- **ステートレス**：サーバーは前のリクエストを覚えない。毎回初対面。
- ログイン状態の維持は HTTP ではなく Cookie やセッションで補っている

## 自分の言葉で説明する
HTTP は Hyper Text Transfer Protocol の略で、クライアントとサーバーが Web 上でデータをやり取りするための通信規約。リクエストとレスポンスの一往復で成り立ち、ステートレス（サーバーが前のやり取りを覚えない）という特徴がある。

## なぜそうなっているのか（Why）
ブラウザとサーバーは別の会社・別のチームが作っている。それでも通信できるのは、HTTP という共通のルールに双方が従っているから。

ステートレスにしているのは設計をシンプルに保つため。状態を持つとサーバー側の管理が複雑になる。

## 詰まったポイント・誤解していたこと
- Transfer を Transform と覚えていた
- 「HTTP リクエストがあるから HTTP リクエストができる」という循環した理解になっていた
- HTTP 自体がプロトコル（ルール）であるという認識が薄かった
